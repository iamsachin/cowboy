<feature>
  <name>Feature 1: useCollapseState composable</name>
  <files>packages/frontend/src/composables/useCollapseState.ts, packages/frontend/tests/composables/useCollapseState.test.ts</files>
  <behavior>
    - isExpanded('id-1') returns false when no state set (default collapsed)
    - toggle('id-1') makes isExpanded('id-1') return true
    - toggle('id-1') twice returns to false
    - expandAll(['id-1', 'id-2', 'id-3']) makes all three return true from isExpanded
    - collapseAll() makes all previously expanded IDs return false
    - expandedCount returns 0 initially, increments on toggle, returns correct count after expandAll
    - collapseAll() resets expandedCount to 0
    - Multiple composable instances are independent (not shared singleton)
  </behavior>
  <implementation>
    Create `useCollapseState()` composable using `reactive(new Map<string, boolean>())`.

    Exports: `isExpanded(id: string): boolean`, `toggle(id: string): void`, `expandAll(ids: string[]): void`, `collapseAll(): void`, `expandedCount: ComputedRef<number>`.

    Key detail: `isExpanded` checks `map.get(id) === true` (absent = collapsed). `collapseAll` calls `map.clear()` (clearing = all collapsed). `expandedCount` is a computed that iterates map values and counts truthy ones.

    Use `reactive(new Map())` NOT `ref(new Map())` -- Vue 3 intercepts Map operations on reactive() for dependency tracking.
  </implementation>
</feature>

<feature>
  <name>Feature 2: Turn helper utilities</name>
  <files>packages/frontend/src/utils/turn-helpers.ts, packages/frontend/tests/utils/turn-helpers.test.ts</files>
  <behavior>
    getPreviewSnippet:
    - Returns first line of assistant text, truncated to 80 chars with '...' if longer
    - Strips XML tags from content before extracting snippet (use stripXmlTags)
    - Falls back to "Used Read, Edit, Bash (3 tool calls)" when text is empty/null
    - Falls back to "Empty response" when no text and no tool calls
    - Handles content with only XML tags (empty after stripping)

    calculateDuration:
    - Returns formatted duration string when tool calls exist (last TC createdAt + TC duration - assistant createdAt)
    - Returns formatted duration from next turn timestamp when no tool calls
    - Returns null when no tool calls and no next turn
    - Returns '--' for negative durations
    - formatMs: <1000 -> "Xms", 1000-59999 -> "Xs", 60000+ -> "Xm Ys"

    truncateOutput:
    - Returns { text: string, truncated: boolean }
    - Text with <= 20 lines returns full text, truncated=false
    - Text with > 20 lines returns first 20 lines, truncated=true
    - Handles null/undefined input gracefully (returns empty string)
    - Uses JSON.stringify(value, null, 2) for non-string values
  </behavior>
  <implementation>
    Create `turn-helpers.ts` with four exports:

    1. `getPreviewSnippet(turn: AssistantTurn): string` -- try stripXmlTags on content, take first line, truncate to 80 chars. Fallback to tool summary, then "Empty response".

    2. `calculateDuration(turn: AssistantTurn, nextTurn?: Turn): string | null` -- compute ms from assistant timestamp. If tool calls exist, use last TC createdAt + (TC duration ?? 0). Else use next turn timestamp. Return null if neither available. Pass ms through formatMs.

    3. `formatMs(ms: number): string` -- format milliseconds to human-readable string.

    4. `truncateOutput(value: unknown, maxLines?: number): { text: string; truncated: boolean }` -- stringify if not string, split by newlines, truncate at maxLines (default 20).

    Import `stripXmlTags` from `../utils/content-sanitizer`. Import `AssistantTurn, Turn` from `../composables/useGroupedTurns`.
  </implementation>
</feature>
