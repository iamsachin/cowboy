  <action>
Create `src-tauri/src/plans.rs` with all 5 plan endpoints. Co-located handlers + queries per user decision.

**pub fn routes() -> Router<AppState>:**
Register 5 routes -- note axum resolves static vs dynamic segments correctly, but register for clarity:
- `/api/plans/stats` (plan_stats)
- `/api/plans/timeseries` (plan_timeseries)
- `/api/plans/by-conversation/:conversationId` (plans_by_conversation)
- `/api/plans/:id` (plan_detail)
- `/api/plans` (plan_list)

**1. plan_list (port getPlanList, plans.ts lines 11-91):**
- Query params: DateRangeParams + PaginationParams + project: Option<String>, status: Option<String>
- INNER JOIN conversations on plans.conversation_id = conversations.id
- Date filter on conversations.created_at with T23:59:59Z suffix
- Optional filters: agent, project, status
- Sort column mapping: date -> plans.created_at, title -> plans.title, steps/totalSteps -> plans.total_steps, completedSteps -> plans.completed_steps, status -> plans.status, agent -> conversations.agent, project -> conversations.project
- ORDER BY sortColumn ASC/DESC, LIMIT/OFFSET pagination
- Total count with same filters
- Response: PlanListResponse { rows: Vec<PlanRow>, total, page, limit }
- PlanRow: { id, conversationId, title, totalSteps, completedSteps, status, createdAt, agent, project: Option<String> }

**2. plan_detail (port getPlanDetail, plans.ts lines 97-156):**
- Path param: plan ID
- Query plan + JOIN conversations for agent/project/conversationTitle + sourceMessageId
- If not found, return AppError::NotFound
- Query plan_steps WHERE plan_id = id, ORDER BY step_number
- Response: PlanDetailResponse { plan: PlanRow, steps: Vec<PlanStepRow>, conversationTitle: Option<String>, sourceMessageId: String }
- PlanStepRow: { id, planId, stepNumber, content, status }

**3. plan_stats (port getPlanStats, plans.ts lines 162-195):**
- Query: COUNT(*), COALESCE(SUM(total_steps), 0), COALESCE(SUM(completed_steps), 0), COALESCE(AVG(total_steps), 0)
- JOIN conversations for date/agent filter
- completionRate = (completedSteps / totalSteps) * 100, 0 if totalSteps is 0
- Round to 1 decimal: (rate * 10).round() / 10.0
- Response: PlanStatsResponse { totalPlans, totalSteps, completionRate, avgStepsPerPlan }

**4. plan_timeseries (port getPlanTimeSeries, plans.ts lines 201-245):**
- Query params: DateRangeParams + granularity: Option<String>
- Auto-granularity same as analytics (port or reuse the function -- can duplicate the small helper in this module or import from analytics if it was made pub)
- strftime period grouping, SUM total_steps, SUM completed_steps, COUNT plans
- Per-period completion rate = (completed/total)*100, round to 1 decimal
- Response: Vec<PlanTimeSeriesPoint> { period, planCount, completionRate }

**5. plans_by_conversation (port getPlansByConversation, plans.ts lines 251-307):**
- Path param: conversationId
- Fetch all plans for this conversation, JOIN conversations for agent/project
- For each plan, fetch its steps (ORDER BY step_number)
- Response: Vec<ConversationPlanEntry> { plan: PlanRow, steps: Vec<PlanStepRow> }
- NOTE: To avoid N+1 queries, fetch all steps for all plan IDs in one query (WHERE plan_id IN (...)), then group in Rust

Wire plans::routes() into server.rs via merge. Add `mod plans;` to lib.rs.

Auto-granularity function: if analytics.rs already exported it as pub, import it. Otherwise, create a small helper in plans.rs (or move it to extractors.rs as a shared utility).
