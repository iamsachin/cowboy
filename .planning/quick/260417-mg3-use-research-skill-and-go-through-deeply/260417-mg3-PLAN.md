    **Section 4 — Gap Analysis.** Organise gaps by surface:
    - *Conversation detail (inline in timeline)*: e.g. is there a visible "N sub-agents
      spawned" summary at the top? Can the user jump directly to the Nth sub-agent from
      the timeline sidebar? How are failed / unmatched sub-agents signaled?
    - *Conversation list / browser / tray*: are sub-agent conversations discoverable as
      their own row, filterable, or only visible through their parent? (Reference quick 69.)
    - *Overview / dashboard*: is there any aggregate view — e.g. "top 5 sub-agent-heavy
      conversations this week", "most used sub-agent types", cost per sub-agent?
    - *Search / command palette*: can Cmd+F or Cmd+K find a sub-agent by name, description,
      or tool output?
    - *Streaming / live*: does the UI update as sub-agents start and finish, or only after
      ingestion completes?
    - *Parallel execution*: after quick 75, are simultaneously-running sub-agents clearly
      grouped visually?
    - *Cost / token attribution*: can the user see per-sub-agent cost contribution to the
      parent conversation?

    For each gap, cite the file/component that would need to change and describe the
    current observed behavior versus the ideal behavior. Do not propose fixes in this
    section — just describe the gap.

    **Section 5 — Prioritised Improvement Backlog.** Produce 6-10 concrete proposals,
    each structured as:

    ```
    ### IMPROVEMENT-N: [Short verb-first title]
    **Discovery problem solved:** [one sentence]
    **User outcome:** [one sentence, observable]
    **Affected files:** [list of 2-5 paths]
    **Approach sketch:** [2-4 sentences — not a full plan, enough to scope]
    **Size:** [S / M / L based on files touched]
    **Priority:** [P0 / P1 / P2 with one-sentence justification]
    **Dependencies / risks:** [other improvements or decisions this touches; e.g. requires
    schema change, or blocks on streaming work]
    ```

    Priority rubric:
    - **P0**: high user-visible impact, small-to-medium effort, no schema changes needed.
    - **P1**: high impact but requires schema/ingestion changes, or medium impact + small effort.
    - **P2**: nice-to-have polish, or large effort for medium impact.

    **Improvement ideas to consider (not exhaustive — derive more from Section 4):**
    1. Sub-agent overview strip at top of conversation (count + quick-jump chips)
    2. Sub-agent filter/facet in ConversationBrowser beyond just hiding columns
    3. Dedicated sub-agent timeline grouping (parallel siblings as a cluster)
    4. Inline cost + token attribution per sub-agent card
    5. Sub-agent status chip states (running / complete / failed / unmatched) with icons
    6. Command palette entry to jump to Nth sub-agent of current conversation
    7. Overview dashboard widget: top sub-agent-heavy conversations / sub-agent success rate
    8. Improved unmatched sub-agent affordance (why didn't it link? what to do?)
    9. Sub-agent type taxonomy / badges (pull from description or agentId)
    10. Timeline sidebar icons that distinguish sub-agent spawn from regular tool call
