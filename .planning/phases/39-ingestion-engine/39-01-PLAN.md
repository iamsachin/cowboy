  <action>
    1. Port `claude-code-parser.ts` to `claude_code_parser.rs`:
       - `pub async fn parse_jsonl_file(file_path: &str) -> Result<ParseResult, ...>` using tokio BufReader::lines()
       - Skip empty lines and non-JSON lines (increment skipped_lines)
       - Only process "user" and "assistant" line types (skip all others)
       - Capture sessionId from first line that has it
       - Record all timestamps
       - User line processing: extract content (string or ContentBlock array), tool results (ToolResultBlock)
       - Assistant chunk processing: use HashMap<String, ChunkAccumulator> keyed by message.id
       - CRITICAL: Each chunk REPLACES contentBlocks (cumulative, not delta). Only capture usage from final chunk (stop_reason != null)
       - Track earliest timestamp per message
       - After all lines: reconstruct assistant messages from chunkMap
       - Compaction detection: check isCompactSummary bool on user lines

    2. Port `normalizer.ts` to `normalizer.rs`:
       - `pub fn normalize_conversation(parse_result: &ParseResult, project_dir: &str, session_id_override: Option<&str>) -> Option<NormalizedData>`
       - Generate conversation ID: `generate_id(&["claude-code", session_id])` where session_id = override or parse_result.session_id
       - Build tool_result_lookup: HashMap<toolUseId, ToolResultData> with content concatenation via "\n" for duplicates
       - User messages: id = generate_id(&[conv_id, uuid])
       - Assistant messages: id = generate_id(&[conv_id, first_uuid])
       - extract_assistant_content: separate text blocks (concatenate with "") and thinking blocks (join with "\n")
       - Strip system XML tags using SYSTEM_TAG_PATTERN regex: `<\/?(system-reminder|local-command-caveat|local-command-stdout|command-name|command-message|command-args|antml:[a-z-]+)(?:\s[^>]*)?\s*\/?>`
       - Tool calls: id = generate_id(&[conv_id, tool_use_id]), status from tool_result lookup (ExitPlanMode+error = "rejected")
       - Token usage: id = generate_id(&[conv_id, message_id_from_api]), messageId = generate_id(&[conv_id, first_uuid])
       - Compaction events: id = generate_id(&[conv_id, "compaction", uuid]), use strip_compaction_preamble and compute_token_delta
       - Model: most common from assistant messages, fallback to token usage
       - Title: derive using should_skip_for_title, XML stripping fallback, assistant content fallback

    3. Create a small test JSONL fixture (2-3 user messages, 2-3 assistant messages with multi-chunk, one tool call, one tool result) in the test module. Verify parse + normalize round-trip produces expected record counts and IDs.
