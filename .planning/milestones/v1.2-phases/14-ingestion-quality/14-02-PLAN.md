  <files>packages/backend/src/ingestion/migration.ts, packages/backend/tests/ingestion/migration.test.ts</files>
  <behavior>
    - fixConversationTitles: conversation with title "Caveat: The messages below..." gets new title from first real user message
    - fixConversationTitles: conversation with title "[Request interrupted by user for tool use]" gets new title
    - fixConversationTitles: conversation with title "/clear" gets new title
    - fixConversationTitles: conversation with NULL title gets title computed from its messages
    - fixConversationTitles: conversation with title "How do I fix this bug?" is NOT modified (already correct)
    - fixConversationModels: claude-code conversation with NULL model gets model from most common token_usage model
    - fixConversationModels: cursor conversation with model "default" gets resolved to "unknown" or actual model from messages
    - fixConversationModels: conversation with model "claude-sonnet-4-6" is NOT modified
    - runDataQualityMigration: runs both fixes, is idempotent
  </behavior>
  <action>
Create `packages/backend/src/ingestion/migration.ts` with:

1. `needsTitleFix(title: string | null): boolean` - Returns true if title is:
   - NULL
   - Matches bad patterns: starts with "Caveat:", "[Request interrupted", "/", or "<" (XML)
   - Empty or whitespace-only

2. `fixConversationTitles(database: typeof db): number` -
   - Query all conversations. For each where `needsTitleFix(title)`:
     - Query its user messages (role='user') ordered by createdAt
     - Apply `shouldSkipForTitle` to find first valid user message content
     - If found, update title (truncated to 100 chars)
     - If not found, try XML-strip fallback (same as normalizer logic)
     - If not found, query assistant messages for first non-null text content, truncate to 100
     - If still nothing, leave as NULL (don't make worse)
   - Return count of updated conversations

3. `fixConversationModels(database: typeof db): number` -
   - Query conversations where model IS NULL AND agent='claude-code':
     - Query token_usage for that conversationId, count model frequency, update with most common
   - Query conversations where model = 'default' AND agent='cursor':
     - Query messages for that conversationId where role='assistant' and model is not null and model != 'default'
     - Count model frequency, use most common. If none found, set to 'unknown'
   - Also fix per-message model for cursor: update messages where model='default' to 'unknown' (for cursor conversations)
   - Return count of updated conversations

4. `runDataQualityMigration(database: typeof db): { titlesFixed: number; modelsFixed: number }` -
   - Call fixConversationTitles and fixConversationModels
   - Log results
   - Return counts

Use Drizzle ORM for all queries (per project convention). Use `eq`, `isNull`, `and`, `sql` from drizzle-orm as needed. Batch updates in transactions.

Write tests using an in-memory SQLite database (create test db, seed with known bad data, run migration, verify). Use `better-sqlite3` + `drizzle-orm` to set up test db with the schema.

   ```typescript
     if (migrationResult.titlesFixed > 0 || migrationResult.modelsFixed > 0) {
       app.log.info({ migrationResult }, 'Data quality migration applied');

   }
   ```
