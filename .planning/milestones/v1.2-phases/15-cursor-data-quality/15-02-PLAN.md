  <action>
    **New function `fixCursorProjects(database, cursorDbPath?)`:**
    1. Query conversations where `agent='cursor' AND project='Cursor'` (literal string match).
    2. If `cursorDbPath` is provided, call `parseCursorDb(cursorDbPath)` to get conversation metadata with workspace paths.
    3. Build a map of `composerId -> workspacePath` from the parsed conversations.
    4. For each DB conversation with project='Cursor', find matching composerData by reverse-engineering the conversation ID (the conversation.id was generated via `generateId('cursor', composerId)` -- so we need to match by iterating cursor conversations and checking if `generateId('cursor', conv.composerId)` matches the DB conversation ID).
    5. If a workspace path is found, extract the last directory segment (basename) as the project name using the same `path.basename()` pattern as `deriveProjectName`.
    6. Update the conversation's project field.
    7. If no cursorDbPath or no match found, leave as 'Cursor' (already a reasonable fallback per user decision).

    **New function `fixCursorMessageContent(database, cursorDbPath?)`:**
    1. Query assistant messages where `conversationId` belongs to a cursor conversation AND `content IS NULL OR content = ''`.
    2. If cursorDbPath is available, attempt to re-read bubble data for those conversations and re-extract content using the same logic as the updated normalizer from plan 15-01.
    3. If content can be derived (from richText, rawText, or other fields), update the message.
    4. If still empty, set content to a tool activity summary (e.g., "Executed tool call") -- same pattern as normalizer.
    5. If no cursorDbPath available, set remaining null-content messages to "Executed tool call" as a reasonable fallback.

    **Update `runDataQualityMigration()`:**
    1. Import `discoverCursorDb` from `cursor-file-discovery.js`.
    2. After existing title and model fixes, call `fixCursorProjects(database, cursorDbPath)` and `fixCursorMessageContent(database, cursorDbPath)` where `cursorDbPath = discoverCursorDb()`.
    3. Extend return type to include `cursorProjectsFixed: number` and `cursorMessagesFixed: number`.

    **Idempotency:** Both functions check condition before fixing (project='Cursor', content IS NULL). Already-fixed records won't match the WHERE clause on subsequent runs.

    **Note on feasibility (per user decision -- Claude's discretion):** Re-extracting message content from Cursor DB is feasible because the bubble data is still in the DB and the parser can re-read it. The migration just needs to map DB message IDs back to bubble IDs, which is possible via the deterministic ID generation (`generateId(conversationId, bubbleId)`).
