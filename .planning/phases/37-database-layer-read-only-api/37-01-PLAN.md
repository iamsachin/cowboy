<task type="auto">
  <name>Task 2: Port conversation list and detail endpoints to Rust</name>
  <files>src-tauri/src/conversations.rs, src-tauri/src/server.rs, src-tauri/src/lib.rs</files>
  <action>
Create `src-tauri/src/conversations.rs` implementing both conversation endpoints with co-located route handlers and query logic. This is the most complex module -- follow the Node.js analytics.ts lines 262-806 exactly.

**pub fn routes() -> Router<AppState>:**
Register two routes:
- `/api/analytics/conversations/:id` (conversation_detail) -- IMPORTANT: In axum, static segments take priority over dynamic, but register detail route for clarity
- `/api/analytics/conversations` (conversation_list)

**conversation_list handler (port getConversationList from analytics.ts lines 262-583):**
Query params struct ConversationListParams merging DateRangeParams + PaginationParams + project: Option<String>, search: Option<String>. Use axum Query<ConversationListParams>.

Port ALL of this logic faithfully:
1. Date range resolution with T23:59:59Z suffix on `to` date
2. Agent/project filters as SQL WHERE conditions
3. Search: LIKE on title/project/model + subquery on messages.content for matching conversation IDs. Build the search snippet with `<mark>` tags and `...` ellipsis (port extractSnippet function).
4. Exclude children (parent_conversation_id IS NULL) from main query
5. hasAssistant subquery: EXISTS (SELECT 1 FROM messages WHERE conversation_id = c.id AND role = 'assistant')
6. LEFT JOIN token_usage, GROUP BY conversations.id
7. Sort column mapping: date -> created_at, agent, project, model, title, inputTokens -> SUM(input+output), cacheReadTokens -> SUM(cache_read)
8. NULLS LAST for nullable columns (agent, project, model, title) via CASE WHEN IS NULL THEN 1 ELSE 0 END
9. Cost sort: fetch ALL rows (no LIMIT/OFFSET), compute costs in Rust, sort, then slice for pagination
10. Normal sort: SQL LIMIT/OFFSET
11. Total count query with same filters (for pagination metadata)
12. Per-model token query for cost calculation: GROUP BY conversation_id, model -- use IN clause with current page conv IDs
13. Children: fetch child conversations (parent_conversation_id IN parent IDs on current page), compute their costs too, group by parent
14. isActive: CASE WHEN status = 'active' THEN 1 ELSE 0 END -> bool
15. hasCompaction: EXISTS subquery on compaction_events -> bool
16. Response: ConversationListResponse { rows, total, page, limit }
17. When search is present, response rows include `snippet` field -- return SearchConversationListResponse { rows, total, page, limit, query }

Use a single db.call() closure for the main query + count + per-model tokens. Use a second db.call() for children + their per-model tokens (to keep closure sizes reasonable).

**Serde structs needed (all with rename_all = "camelCase"):**
- ConversationRow (id, date, agent, title, project, model, inputTokens, outputTokens, cacheReadTokens, cacheCreationTokens, cost, savings, isActive, hasCompaction, parentConversationId, parentTitle, children: Option<Vec<ConversationRow>>)
- ConversationListResponse (rows: Vec<ConversationRow>, total: i64, page: u32, limit: u32)
- SearchConversationRow (extends ConversationRow + snippet: Option<String>)
- SearchConversationListResponse (rows, total, page, limit, query)

**conversation_detail handler (port getConversationDetail from analytics.ts lines 629-806):**
Path param: conversation ID string.

Port ALL of this logic in a single db.call() closure (all reads, same thread):
1. Fetch conversation row or return AppError::NotFound
2. Fetch messages ordered by created_at
3. Fetch tool_calls ordered by created_at -- parse subagent_summary TEXT column with serde_json::from_str into SubagentSummary struct
4. Token summary: GROUP BY model, sum all token columns, compute cost per model, total
5. Per-message token usage: GROUP BY message_id (WHERE message_id IS NOT NULL)
6. Compaction events ordered by timestamp
7. firstMessageAt/lastMessageAt from first/last message, falling back to conversation createdAt/updatedAt
8. Parent title lookup if parentConversationId is set

**Serde structs needed for detail:**
- ConversationDetailResponse, MessageRow, ToolCallRow, SubagentSummary, MessageTokenUsage, CompactionEvent
- Match packages/shared/src/types/api.ts lines 76-141 exactly

**Critical correctness notes:**
- SQLite boolean: read as i64, convert to bool (row.get::<_, i64>(idx)? == 1)
- COALESCE(SUM(...), 0) on all aggregate queries to avoid NULL
- tool_calls.input and tool_calls.output are TEXT columns storing JSON -- read as Option<String>, deserialize with serde_json::from_str to serde_json::Value
- token_usage per-message: model column needed for cost calculation per message
- Cost null handling: if no pricing match, cost is None/null in JSON (not 0)
- hasCost flag in tokenSummary: if no model matched pricing, cost and savings are null

Wire into server.rs: `app.merge(conversations::routes())`. Update AppState type alias: `pub type AppState = Arc<Connection>;` and export it so domain modules can import it.

Update lib.rs: add `mod conversations;`
