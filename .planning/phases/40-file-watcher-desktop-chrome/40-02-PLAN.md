  <action>
1. **Tray icon asset:** Create a 44x44px (22x22pt Retina) monochrome PNG at `src-tauri/icons/tray-icon.png`. This should be a simple cowboy hat silhouette -- black shape on transparent background. Use the simplest approach available (e.g., create programmatically with the `image` crate in a build script, or use a minimal hand-crafted PNG). The icon MUST be a template image (black on transparent) so macOS auto-inverts for dark mode. If creating a PNG programmatically is complex, a simple placeholder silhouette is acceptable -- the user has a custom app icon and can refine the tray icon later.

2. **lib.rs -- Native menu bar:** In the `.setup()` closure, before spawning the server:
   - Build the "Cowboy" app submenu with `SubmenuBuilder::new(app, "Cowboy")`:
     - `.about(None::<tauri::menu::AboutMetadata>)` -- standard About dialog
     - `.separator()`
     - `.quit()` -- standard Quit with Cmd+Q
   - Build the "Edit" submenu with `SubmenuBuilder::new(app, "Edit")`:
     - `.undo()`, `.redo()`, `.separator()`, `.cut()`, `.copy()`, `.paste()`, `.select_all()`
   - Build the menu with `MenuBuilder::new(app).items(&[&app_menu, &edit_menu]).build()?`
   - Set it with `app.set_menu(menu)?`

3. **lib.rs -- System tray:** Still in `.setup()`:
   - Create menu items: `MenuItem::with_id(app, "show", "Show", true, None::<&str>)?` and `MenuItem::with_id(app, "quit", "Quit", true, None::<&str>)?`
   - Build tray menu: `Menu::with_items(app, &[&show_item, &quit_item])?`
   - Load icon: `Image::from_path("icons/tray-icon.png")` -- the path is relative to the `src-tauri/` directory. If `from_path` doesn't work with relative paths at runtime, use `include_bytes!("../icons/tray-icon.png")` with `Image::from_bytes(...)` instead.
   - Build tray: `TrayIconBuilder::new().icon(tray_icon).icon_as_template(true).menu(&tray_menu).on_menu_event(...)`.build(app)?`
   - In `on_menu_event`: match on `"show"` to get webview window "main", call `.show()` and `.set_focus()`. Match on `"quit"` to call `app.exit(0)`.

4. **lib.rs -- Close-to-tray:** Add `.on_window_event()` to the Tauri builder (AFTER `.setup()`, BEFORE `.run()`):
   ```rust
   .on_window_event(|window, event| {
       if let WindowEvent::CloseRequested { api, .. } = event {
           let _ = window.hide();
           api.prevent_close();
       }
   })
   ```
   This makes the red X button hide the window instead of closing the app. The app only exits via tray Quit or native menu Quit (Cmd+Q).

5. **Keep existing setup logic intact:** The database initialization and server spawn remain unchanged. Add the menu/tray code BETWEEN db init and server spawn.
