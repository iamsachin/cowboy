<objective>
Index sub-agent synthesized content (description, prompt first line, last_error, files_touched, tool_breakdown keys) into a dedicated SQLite FTS5 virtual table (`subagent_fts`) so searching in `ConversationBrowser` finds parent conversations whose sub-agents match. Backfill existing DBs on first startup after ship.

Purpose: IMPR-6 from FINDINGS §5. Research confirmed (§3.4 #8, §4.4b, §4.4c) that **no FTS virtual table exists in the schema today** — search is LIKE-based over `messages.content` + `conversations.title/project/model` (`conversations.rs:301-309`). `subagent_summary` JSON is invisible. This plan creates the FTS infrastructure specifically for sub-agents.

Output:
- `subagent_fts` FTS5 virtual table, columns `content` (tokenized) + `tool_call_id UNINDEXED` + `kind UNINDEXED` (always `'subagent'` for forward compatibility).
- Idempotent schema creation in `schema.sql` and idempotent migration + backfill in `db.rs`.
- Ingestion-time INSERT OR REPLACE at the existing summary UPDATE site.
- Search handler extension with subquery union + origin-aware snippet.
</objective>
