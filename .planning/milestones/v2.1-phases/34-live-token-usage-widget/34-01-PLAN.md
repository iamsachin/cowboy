    - Test 5: getTokenRate returns results sorted by minute ascending

  <action>
    1. In `packages/shared/src/types/api.ts`, add the TokenRatePoint interface at the end (before Plan types section):
       ```typescript
       export interface TokenRatePoint {
         minute: string;        // e.g. "2026-03-10T14:23"
         inputTokens: number;
         outputTokens: number;
       }
       ```
    2. In `packages/shared/src/types/index.ts`, add `TokenRatePoint` to the re-export line from `./api.js`.
    3. In `packages/backend/src/db/queries/analytics.ts`, add a `getTokenRate()` function:
       - Import `TokenRatePoint` from `@cowboy/shared`
       - Query tokenUsage table directly (no conversation join needed -- all agents combined per user decision)
       - WHERE `createdAt >= datetime('now', '-60 minutes')`
       - GROUP BY `strftime('%Y-%m-%dT%H:%M', createdAt)`
       - SELECT minute, coalesce(sum(inputTokens), 0), coalesce(sum(outputTokens), 0)
       - ORDER BY minute ASC
       - Map rows to `TokenRatePoint[]` with Number() coercion
       - Export the function
    4. Create `packages/backend/tests/analytics/token-rate.test.ts` with vitest tests covering the 5 behaviors above. Follow existing test patterns in the analytics test directory (check a sibling test for setup/db patterns).
