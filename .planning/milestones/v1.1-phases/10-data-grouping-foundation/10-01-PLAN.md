<feature>
  <name>Turn Grouping Algorithm</name>
  <files>packages/frontend/src/composables/useGroupedTurns.ts, packages/frontend/src/utils/content-parser.ts, packages/frontend/tests/composables/useGroupedTurns.test.ts</files>
  <behavior>
    - groupTurns([user msg], []) -> [{ type: 'user', message: userMsg }]
    - groupTurns([assistant msg], [tc with matching messageId]) -> [{ type: 'assistant', message: assistantMsg, toolCalls: [tc] }]
    - groupTurns([user, assistant], [tc1, tc2 both linked to assistant]) -> [userTurn, assistantTurn with 2 toolCalls sorted by createdAt]
    - groupTurns([assistant1, assistant2], []) -> 2 separate assistant turns (never merged)
    - groupTurns([assistant], [orphan tc with wrong messageId]) -> orphan attached to assistant turn (nearest preceding by timestamp)
    - groupTurns([user, assistant], [orphan before any assistant]) -> orphan dropped (no preceding assistant turn)
    - groupTurns([assistant with null content], [tc linked to it]) -> assistant turn still created (tool calls are valuable context)
    - groupTurns([], []) -> []
    - Tool calls within a turn are sorted by createdAt ascending
    - Messages are sorted by createdAt regardless of input order
  </behavior>
  <implementation>
    RED: Create test file with all behavior cases above using vitest. Tests import groupTurns and Turn types. Use factory helpers to create MessageRow/ToolCallRow fixtures with minimal required fields. Run tests -- all must fail.

    GREEN: Implement groupTurns in useGroupedTurns.ts following the algorithm from RESEARCH.md:
    1. Build Map<messageId, ToolCallRow[]> from toolCalls array
    2. Identify orphans (messageId not in message set)
    3. Sort messages by createdAt
    4. Walk messages: user -> UserTurn, assistant -> AssistantTurn with linked tool calls
    5. Sort tool calls within each turn by createdAt
    6. Attach orphans to nearest preceding assistant turn by timestamp
    7. Export Turn, UserTurn, AssistantTurn types

    Also extract from ChatMessage.vue into content-parser.ts:
    - parseContent(raw: string): ContentBlock[] -- the code block parsing logic
    - formatTime(isoString: string): string -- the time formatter
    - ContentBlock interface

    REFACTOR: Ensure groupTurns is a pure function (no Vue reactivity). The composable wrapper (computed) will be added in Plan 02 when wiring to ConversationDetail.

    Note on empty messages: Do NOT filter out empty assistant messages in groupTurns. An assistant message with null/empty content but associated tool calls should still produce a turn. The isEmptyMessage filtering moves to the UI layer (skip rendering text if empty, but still show tool calls).
  </implementation>
