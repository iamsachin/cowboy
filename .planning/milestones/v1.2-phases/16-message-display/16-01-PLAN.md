<objective>
Extend the turn grouping model to represent system-injected messages, slash commands, and /clear dividers as distinct turn types instead of filtering them out.

Purpose: The current `groupTurns` function only produces `UserTurn` and `AssistantGroup` types. System messages are filtered upstream in ConversationDetail.vue. This plan adds new turn types so the UI layer (Plan 02) can render them with distinct styling.

Output: Extended `useGroupedTurns.ts` with new types and a `classifySystemMessage` helper, plus comprehensive tests.
</objective>

<feature>
  <name>Extend turn grouping with system message and slash command types</name>
  <files>packages/frontend/src/composables/useGroupedTurns.ts, packages/frontend/tests/composables/useGroupedTurns.test.ts</files>
  <behavior>
    New types to add:
    - `SystemMessageCategory`: union type of 'system-reminder' | 'skill-instruction' | 'objective' | 'system-caveat' | 'other'
    - `SystemGroup`: { type: 'system-group', messages: MessageRow[], categories: SystemMessageCategory[], count: number }
    - `SlashCommandTurn`: { type: 'slash-command', message: MessageRow, commandText: string }
    - `ClearDividerTurn`: { type: 'clear-divider', message: MessageRow }
    - Updated `GroupedTurn` union includes these new types

    New helper:
    - `classifySystemMessage(content: string): SystemMessageCategory` — examines content to determine category:
      - Contains `<system-reminder` → 'system-reminder'
      - Contains `<objective>` or `<execution_context>` → 'objective'
      - Contains `<context>` with `<files_to_read>` or `<process>` → 'skill-instruction'
      - Contains `Caveat:` or starts with system caveat pattern → 'system-caveat'
      - Otherwise → 'other'

    Updated `groupTurns` behavior:
    - Accept ALL user messages (no longer assumes system messages are pre-filtered)
    - For each user message, classify:
      1. `isClearCommand(content)` → produce ClearDividerTurn
      2. `isSlashCommand(content)` → produce SlashCommandTurn with extracted command text
      3. `isSystemInjected(content)` → accumulate into pending system group
      4. Otherwise → flush pending system group, produce UserTurn
    - Consecutive system-injected messages are merged into one SystemGroup
    - SystemGroup carries array of categories (one per message) for UI labeling
    - Non-system user messages flush the pending system group (just like they flush pending assistant turns)

    Test cases:
    - classifySystemMessage returns correct category for each pattern
    - Single system message produces SystemGroup with count=1
    - Three consecutive system messages produce one SystemGroup with count=3 and correct categories
    - System messages between user and assistant are grouped, not lost
    - Slash command `/gsd:execute-phase 11` produces SlashCommandTurn with commandText="/gsd:execute-phase 11"
    - `/clear` produces ClearDividerTurn (not SlashCommandTurn)
    - Mixed sequence: system, system, user, assistant → SystemGroup + UserTurn + AssistantGroup
    - Regular user messages still produce UserTurn (no regression)
    - Existing test suite still passes (no changes to assistant grouping logic)
  </behavior>
  <implementation>
    1. Add new type interfaces to useGroupedTurns.ts above existing types
    2. Export `classifySystemMessage` function
    3. Update `GroupedTurn` union type to include new types
    4. Modify `groupTurns` to handle user message classification before creating UserTurn
    5. Add a `pendingSystem: MessageRow[]` accumulator similar to `pendingAssistant`
    6. Add `flushSystem()` helper that creates a SystemGroup from accumulated messages
    7. Import `isSystemInjected`, `isSlashCommand`, `isClearCommand`, `extractCommandText` from content-sanitizer
  </implementation>
</feature>
