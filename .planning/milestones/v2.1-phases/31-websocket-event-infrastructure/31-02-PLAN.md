  <action>
1. Rewrite `packages/frontend/src/composables/useWebSocket.ts`:
   - Import `WebSocketEvent`, `WebSocketEventType`, `SystemFullRefreshEvent` from `@cowboy/shared`
   - Import `getCurrentScope`, `onScopeDispose` from `vue`
   - Keep module-level singleton pattern (state, reconnectAttempt, ws, reconnectTimer, started refs)
   - Replace `const listeners = new Set<() => void>()` with `const listeners = new Map<string, Set<(event: any) => void>>()`
   - Add `let lastSeq = 0` for sequence tracking
   - Create typed `on()` function:
     ```typescript
     type EventCallback<T extends WebSocketEventType> = (event: Extract<WebSocketEvent, { type: T }>) => void;

     function on<T extends WebSocketEventType>(type: T, callback: EventCallback<T>): () => void {
       if (!listeners.has(type)) listeners.set(type, new Set());
       listeners.get(type)!.add(callback as any);
       const unsubscribe = () => { listeners.get(type)?.delete(callback as any); };
       if (getCurrentScope()) { onScopeDispose(unsubscribe); }
       return unsubscribe;
     }
     ```
   - Rewrite `ws.onmessage` handler:
     - Parse JSON as `WebSocketEvent`
     - Skip messages without `seq` field (e.g. the `connected` handshake)
     - Gap detection: if `msg.seq > lastSeq + 1 && lastSeq > 0`, create synthetic `SystemFullRefreshEvent` and fire it to `system:full-refresh` listeners
     - Update `lastSeq = msg.seq`
     - Route to type-specific listeners: `listeners.get(msg.type)?.forEach(cb => cb(msg))`
   - On reconnect (in `ws.onopen`): reset `lastSeq = 0` so first event after reconnect sets new baseline
   - Update `handleVisibilityChange`: when tab becomes visible, instead of `listeners.forEach(cb => cb())`, create synthetic `system:full-refresh` event and fire it to all `system:full-refresh` listeners
   - Remove `onDataChanged` from the returned API â€” replace with `on`
   - Return: `{ state, reconnectAttempt, on }`
   - Update `_resetForTesting` to also clear the listeners Map and reset lastSeq

2. Remove the `WsMessage` type (no longer needed â€” we use `WebSocketEvent` from shared)
   - Keep `ConnectionState` export (still used by ConnectionStatus.vue)

3. Update `packages/frontend/tests/composables/useWebSocket.test.ts`:
   - Replace all `onDataChanged` tests with `on(type, callback)` tests
   - Test: `on('conversation:changed', cb)` fires cb when a conversation:changed message arrives
   - Test: `on('conversation:created', cb)` fires cb when conversation:created arrives
   - Test: `on('system:full-refresh', cb)` fires cb when system:full-refresh arrives
   - Test: callback receives the full typed event object (not just called with no args)
   - Test: gap detection â€” send events with seq 1, then seq 5, verify system:full-refresh fires
   - Test: on reconnect (ws.onopen), lastSeq resets â€” send seq 1 after reconnect without triggering gap
   - Test: unsubscribe works â€” after calling returned function, callback no longer fires
   - Test: multiple listeners on same type all fire
   - Test: listeners on different types only receive their type
