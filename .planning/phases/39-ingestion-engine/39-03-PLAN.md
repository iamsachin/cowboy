    - get_bubbles_for_conversation queries "bubbleId:{composerId}:*" ordered by rowid ASC

    1. Port `cursor-file-discovery.ts` to `cursor_file_discovery.rs`:
       - `pub fn discover_cursor_db() -> Option<String>`
       - macOS: `dirs::home_dir()/.../Library/Application Support/Cursor/User/globalStorage/state.vscdb`
       - Check file exists with std::fs::metadata, return None if not found
       - Only support macOS for v3.0 (match Node.js darwin check)

    2. Port `cursor-parser.ts` to `cursor_parser.rs`:
       - Define CursorConversation and CursorBubble structs
       - CursorConversation: composerId, name, createdAt (f64), lastUpdatedAt (f64), status, isAgentic, usageData (serde_json::Value), modelConfig (serde_json::Value), fullConversationHeadersOnly (serde_json::Value), workspacePath
       - CursorBubble: bubbleId, bubble_type (u32), text, createdAt (Option<serde_json::Value>), tokenCount, modelInfo, timingInfo, toolFormerData (serde_json::Value), isCapabilityIteration, capabilityType, tokenCountUpUntilHere, thinking

       - `pub fn parse_cursor_db(db_path: &str) -> Result<Vec<CursorConversation>, ...>`:
         Open with `rusqlite::Connection::open_with_flags(path, OpenFlags::SQLITE_OPEN_READ_ONLY | OpenFlags::SQLITE_OPEN_NO_MUTEX)`
         Query: `SELECT key, CAST(value AS TEXT) as value FROM cursorDiskKV WHERE key LIKE 'composerData:%' AND LENGTH(value) > 100`
         Parse each row's JSON value, extract composerId from key
         workspacePath: try data.workspacePath, then data.workspaceFolder, then data.rootDir, then data.context.workspacePath

       - `pub fn get_bubbles_for_conversation(db_path: &str, composer_id: &str) -> Result<Vec<CursorBubble>, ...>`:
         Query: `SELECT key, CAST(value AS TEXT) as value FROM cursorDiskKV WHERE key LIKE ? ORDER BY rowid ASC`
         Bind: `bubbleId:{composer_id}:%`
         Extract bubbleId from key (last segment after ":")

       IMPORTANT: These use sync rusqlite (not tokio-rusqlite) since vscdb is a separate database opened readonly. When called from async context, wrap in `tokio::task::spawn_blocking`.

  <files>

  <behavior>
    - normalize_cursor_conversation returns None for empty bubbles
    - Conversation ID: generate_id(&["cursor", composer_id])
    - User bubbles emitted directly as individual messages
    - Consecutive assistant bubbles merged into single logical turns
    - Merged content: text segments joined with "\n\n", prefixed with "Executed N tool calls" if tool bubbles present
    - Thinking segments extracted from bubble.thinking.text and joined with "\n\n"
    - Tool calls extracted from toolFormerData with name, params/rawArgs, result, status mapping
    - Status mapping: "completed" -> "success", "error" -> "error", others pass through
    - Model: "default" mapped to "unknown"
    - normalize_cursor_timestamp: null/0 -> now, numeric < 1e12 -> seconds*1000, else -> ms
    - Token usage from per-bubble tokenCount (inputTokens, outputTokens)
    - Cumulative token differential: when tokenCount missing but tokenCountUpUntilHere present, compute delta from previous
    - No compaction events for Cursor (empty vec)
    - Title derived using same skip logic as Claude Code (shouldSkipForTitle)
  </behavior>
  <action>
    Port `cursor-normalizer.ts` to `cursor_normalizer.rs`:

    1. `pub fn normalize_cursor_conversation(conv: &CursorConversation, bubbles: &[CursorBubble], project: &str) -> Option<NormalizedData>`:
       - Return None if bubbles is empty
       - conversation_id = generate_id(&["cursor", &conv.composer_id])
       - title = derive_title(bubbles) using should_skip_for_title, XML strip fallback, assistant text fallback
       - created_at/updated_at from normalize_cursor_timestamp
       - model from derive_model: try modelConfig first (skip "default"), then most common across AI bubbles (skip "default"), if only "default" found return "unknown"

    2. Message merging loop:
       - Iterate bubbles. User (type=1) emitted directly.
       - Assistant (type=2): collect consecutive into group, then merge:
         - thinking_segments from bubble.thinking.text
         - text_segments from bubble.text (only if has text and is not tool-only)
         - Tool-only detection: no text AND no thinking AND (capabilityType != null OR toolFormerData present)
         - Merged content: text joined with "\n\n", optionally prefixed with "Executed N tool call(s)"
         - Tool calls from bubbles with toolFormerData.name: id from toolCallId or bubbleId, input from params/rawArgs (safe JSON parse), output from result (safe JSON parse with string fallback), status mapped
       - Track bubbleToMergedId mapping for token usage

    3. Token usage computation:
       - For each type=2 bubble in order:
         - If tokenCount has inputTokens or outputTokens > 0 and bubble is included: create record
         - Else if tokenCountUpUntilHere > prevCumulative and bubble is included: estimate outputTokens as delta
         - Always update prevCumulativeTokens tracker
       - Token usage ID: generate_id(&[conv_id, format!("token-{}", bubble_id)])

    4. Helper functions:
       - normalize_cursor_timestamp(ts: f64_or_string_or_null) -> String (ISO 8601)
       - derive_bubble_timestamp: try bubble.createdAt, then timingInfo.clientStartTime, then conv.createdAt
       - safe_json_parse: return parsed value or None
       - safe_json_parse_with_fallback: return parsed or raw string
       - map_tool_status: "completed" -> "success", "error" -> "error", others pass through

    5. Update mod.rs to add `pub mod cursor_file_discovery; pub mod cursor_parser; pub mod cursor_normalizer;`
  </action>
